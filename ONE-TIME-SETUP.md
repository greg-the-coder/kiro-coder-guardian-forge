# One-Time Setup for Kiro + Coder (2 minutes)

This guide walks you through verifying your Coder workspace is properly configured for git operations. In most cases, everything is already set up automatically!

---

## What's Automatic (No Setup Required!)

### ✅ Fully Automatic - Coder Handles Everything

Your Coder deployment automatically configures:
- **Git SSH authentication** - Coder's git SSH wrapper (`coder gitssh`) handles all SSH authentication
- **SSH keys** - Managed centrally by Coder (no user SSH keys needed!)
- **Git identity configuration** - Sets your name and email from Coder profile
- **Git remote format** - Uses SSH format (`git@github.com:user/repo.git`)
- **Environment variables** - Sets `GIT_SSH_COMMAND` to use Coder's wrapper

**Important:** The Coder git SSH wrapper (`coder gitssh`) is automatically configured via the `GIT_SSH_COMMAND` environment variable. This wrapper uses Coder-managed SSH keys for authentication, so you don't need to generate, manage, or add SSH keys to GitHub/GitLab!

### 🎉 No Manual Setup Required!

Unlike traditional git workflows, you don't need to:
- ❌ Generate SSH keys (`ssh-keygen`)
- ❌ Add SSH keys to GitHub/GitLab
- ❌ Manage SSH key permissions
- ❌ Configure SSH agents

**Coder handles all of this automatically through its git SSH wrapper!**

## Verification Steps

### Step 1: Verify Coder Git SSH Wrapper

The Coder git SSH wrapper is the key component that makes git operations work automatically.

```bash
echo $GIT_SSH_COMMAND
```

**Expected output:**
```
/tmp/coder.Ir4oL9/coder gitssh --
```

**Note:** The path `/tmp/coder.*/coder` is dynamically generated by Coder and will vary between workspaces.

✅ If you see this, the git SSH wrapper is configured correctly!

---

### Step 2: Test Git Authentication

Verify that git operations work through the Coder wrapper:

```bash
git ls-remote git@github.com:coder/coder.git HEAD
```

**Expected output:**
```
<commit-hash>        HEAD
```

✅ If you see a commit hash, git authentication is working!

**Alternative test:**
```bash
$GIT_SSH_COMMAND -T git@github.com 2>&1
```

**Expected output:**
```
Hi <coder-username>! You've successfully authenticated, but GitHub does not provide shell access.
```

**Note:** The exit code will be 1 (because GitHub doesn't provide shell access), but the authentication message confirms it's working.

---

### Step 3: Verify Git Configuration

Check that your git identity is configured correctly:

```bash
env | grep GIT_
```

You should see:
```
GIT_AUTHOR_NAME=your-name
GIT_AUTHOR_EMAIL=your@email.com
GIT_COMMITTER_NAME=your-name
GIT_COMMITTER_EMAIL=your@email.com
GIT_SSH_COMMAND=/tmp/coder.Ir4oL9/coder gitssh --
```

**Important:** The `GIT_SSH_COMMAND` variable points to Coder's custom git SSH wrapper. This wrapper (`coder gitssh`) handles SSH authentication through Coder's infrastructure using Coder-managed SSH keys. The path `/tmp/coder.*/coder` is dynamically generated by Coder and will vary between workspaces.

✅ If you see these variables (especially `GIT_SSH_COMMAND`), your git configuration is correct!

---

### Step 4: Verify Git Remote Format

If you have an existing repository cloned, verify it uses SSH format:

```bash
cd /workspaces/your-project
git remote -v
```

You should see:
```
origin  git@github.com:user/repo.git (fetch)
origin  git@github.com:user/repo.git (push)
```

✅ If you see `git@github.com:...`, the remote is correctly configured!

❌ If you see `https://github.com/...`, convert to SSH:
```bash
git remote set-url origin git@github.com:user/repo.git
git remote -v  # Verify the change
```

## Troubleshooting

### Problem: GIT_SSH_COMMAND not set

**Symptoms:**
- `echo $GIT_SSH_COMMAND` returns empty
- Git operations fail with "Permission denied (publickey)"

**Cause:** Coder workspace not properly configured

**Solution:**
1. Restart workspace to reload environment variables
2. Check Coder agent logs: Look for git SSH wrapper initialization
3. Contact Coder administrator to verify template configuration

---

### Problem: Git SSH wrapper binary missing

**Symptoms:**
- `GIT_SSH_COMMAND` is set but binary doesn't exist
- Git operations fail with "command not found"

**Cause:** Coder agent not properly initialized

**Solution:**
```bash
# Check if wrapper exists
ls -la $(echo $GIT_SSH_COMMAND | awk '{print $1}')

# If missing:
# 1. Restart workspace
# 2. Check Coder agent logs
# 3. Contact Coder administrator
```

---

### Problem: "Could not resolve hostname"

**Cause:** Network connectivity issue

**Solution:**
1. Check network: `ping github.com`
2. If ping fails, check your internet connection
3. Contact Coder administrator if issue persists

---

### Problem: Git remote uses HTTPS instead of SSH

**Cause:** Repository was cloned with HTTPS URL

**Solution:**
```bash
cd /workspaces/your-project

# Get current remote URL
git remote get-url origin

# Convert HTTPS to SSH
# From: https://github.com/user/repo.git
# To:   git@github.com:user/repo.git
git remote set-url origin git@github.com:user/repo.git

# Verify
git remote -v
```

---

### Problem: "Host key verification failed"

**Cause:** GitHub/GitLab host key not in known_hosts

**Solution:**
```bash
# Add GitHub to known hosts
ssh-keyscan github.com >> ~/.ssh/known_hosts

# Or for GitLab
ssh-keyscan gitlab.com >> ~/.ssh/known_hosts

# Test again
git ls-remote git@github.com:user/repo.git HEAD
```

---

### Problem: Git operations work in home workspace but fail in task workspace

**Cause:** Task workspace may not have inherited `GIT_SSH_COMMAND` environment variable

**Solution:**
```bash
# In task workspace, verify GIT_SSH_COMMAND is set
echo $GIT_SSH_COMMAND

# If empty, contact Coder administrator to update template
# Template must ensure environment variables are inherited by task workspaces
```
