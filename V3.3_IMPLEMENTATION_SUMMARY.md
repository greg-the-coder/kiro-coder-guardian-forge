## Key Features Delivered

### 1. Post-Task Analysis Automation

**Three Analysis Types:**
1. **Consistency Analysis**
   - Component matching across deliverables
   - Data structure alignment checking
   - Visual design consistency validation
   - Scoring system (0-100%)

2. **Requirements Compliance**
   - Functional requirements (FR-X) tracking
   - Non-functional requirements (NFR-X) tracking
   - Traceability matrix generation
   - Evidence mapping

3. **Executive Summary**
   - Project metrics and scores
   - Key findings and recommendations
   - Deployment readiness assessment
   - Stakeholder-ready format

**Generated Reports:**
- `CONSISTENCY_ANALYSIS.md`
- `REQUIREMENTS_COMPLIANCE.md`
- `EXECUTIVE_SUMMARY.md`

### 2. Validation-First Task Creation

**Validation Layers:**
1. **Pre-completion** - Workspace agent validates before finishing
2. **Post-transfer** - External agent verifies after merge
3. **Pre-deployment** - Quality gates before production

**Project Types Supported:**
- Python (flake8, pytest, black, mypy)
- Node.js/TypeScript (ESLint, tests, build, audit)
- Go (gofmt, go vet, go test, race detector)
- React (ESLint, component tests, build, a11y)
- API (REST, security, documentation)
- Infrastructure (Terraform, security, IAM)

### 3. Quality Gates

**Two Gates:**
1. **Pre-merge gate** - Validates before merging feature branch
   - Runs validation checks
   - Verifies tests pass
   - Checks linting
   - Ensures git hygiene

2. **Pre-deployment gate** - Ensures deployment readiness
   - Checks consistency score (min 85%)
   - Validates compliance score (min 90%)
   - Verifies dependencies install
   - Runs test suite

## How to Use

### Quick Start

1. **Create task with validation:**
```python
from validation_patterns import generate_validation_prompt

task_prompt = generate_validation_prompt(
    project_type='python',
    task_description='Implement user authentication'
)

task = coder_create_task(input=task_prompt, ...)
```

2. **After task completes, run analysis:**
```python
from post_task_analysis import complete_post_task_analysis

analysis = complete_post_task_analysis(
    home_workspace=home_workspace,
    repo_path=repo_path
)
```

3. **Check quality gates:**
```python
from validation_patterns import pre_deployment_quality_gate

if pre_deployment_quality_gate(home_workspace, repo_path):
    deploy()
```

### Complete Workflow

See `steering/task-workflow.md` for the complete `complete_validated_task_workflow()` function that integrates all v3.3 capabilities.
